# ADR-0011: Cleaning v2 + SOP v3 Implementation

Status: Implemented (Week 3 Day 1, 2026-05-13 13:00 BST)
Date: 2026-05-13
Context: SOP_v2 baseline benchmark revealed two root causes of metadata pollution:
  1. SOP_v2 prompts explicitly required markdown structure and page references
  2. cleaning_v1 only filtered content_layer, missing references labeled as list_item
This ADR records the joint fix.

---

## Decision

Replace cleaning_v1 with cleaning_v2 (section_header boundary detection) AND
replace SOP_v2 prompts with SOP_v3 (clean-output rules) in a single pipeline.py
edit, deployed Week 3 Day 1 afternoon.

Both changes share one motivation (eliminate fake_page + markdown pollution)
and are validated by one experiment (5 golden papers --force dry run, 
benchmark against SOP_v2 baseline).

---

## Root cause analysis

### Cause 1: SOP_v2 prompts demanded the pollution

scripts/ingest/pipeline.py line 145-183 contained prompts like:

  L1_factual_background: "Extract the following with exact quotes and 
                          page numbers where possible: 1) Research Background
                          2) Core Concepts 3) Common Pitfalls..."
  L1_factual_methods:    "List all methods organized as: A) Experimental 
                          Methods B) Analytical/Modeling Methods 
                          C) Validation Methods. Include page numbers."
  L2_interpretive_problem: "Answer with page references: 1) Most central
                            problem... 2) Hardest difficulties..."

These prompts directly caused:
  - "page numbers" requested 3 times -> 149 fake pages
  - "1) 2) 3)" enumerate format -> markdown numbered lists
  - "A) B) C)" sub-heading -> bold markdown
  - Multi-question structure -> LLM echoes the questions back

### Cause 2: cleaning_v1 missed references

scripts/ingest/pipeline.py line 86-95:
  kept = [t for t in texts if t.get("content_layer") == "body"]

Docling labels references as content_layer="body", label="list_item".
So cleaning_v1 kept them. Result:
  - P117 cleaning dropped only 0.5% (references survived)
  - P1-P5-P44-P77-P117 cleaned.md all end with reference lists
  - SOP_v2 sees "Smith J 2024 ... pp 25-30" in input
  - LLM learns "page citations are normal" -> hallucinates more pages

Verified via docling.json label analysis:
  P117: 51% of body items are list_item (mostly references)
  P77:  33%
  P1:   32%
  P44:  21%
  P5:   19%

## Solution: cleaning_v2

Detects body range using section_header markers:

  BODY_START: first section_header matching ^(abstract|introduction|1\.\s)
  BODY_END:   first section_header (after start) matching ^(references|
              bibliography|acknowledg(e?)ments?|funding|conflict|data 
              availability|author contributions|orcid)
  
  kept = [t in texts[body_start:body_end] if content_layer == "body"]
  dropped = pre_body + post_body + furniture_in_range

Tested on 5 golden papers before deployment:
  P1:    136K -> 98K chars   (28% additional reduction over v1)
  P5:    212K -> 177K chars  (17%)
  P44:    78K -> 50K chars   (35%)
  P77:    81K -> 65K chars   (20%)
  P117:  125K -> 95K chars   (24%)

All 5 papers' new cleaned.md verified to end at real body content 
(no references, no author bios, no TOC entries).

### Provenance addition

cleaning_v2 adds to _cleaning block:
  body_range: [start_idx, end_idx]
  dropped_pre_body: int (TOC + author affiliations dropped)
  dropped_post_body: int (references + acknowledgments dropped)
  dropped_furniture_in_range: int (page headers/footers)

## Solution: SOP v3 prompts

Five prompts rewritten with:
  1. CRITICAL OUTPUT RULES block prohibiting markdown / fake pages / filler
  2. No enumerate or sub-heading demands
  3. "continuous narrative" / "continuous prose" task descriptions
  4. L2_scoring has explicit exception: one-line-per-dimension is the ONLY 
     allowed structured list, all else is prose
  5. Direct "Output:" cue at the end (no preamble expected)

Token cost: SOP_v3 prompt ~3.2K chars (vs SOP_v2 ~1.6K), shared GLOBAL_RULES
block repeated 5 times. Future optimization: move to system message if 
Ollama supports it.

### Excluded from SOP v3 (deferred to Week 4 / Week 5)

  - verbatim_quote field (ADR-0010 Layer 3) - deferred until cleaning_v2 
    stabilizes, then add quote-validation post-processor
  - char_offset_in_chunk field - needs chunk metadata schema v2
  - per-domain SOP variants (ADR-0009) - deferred to Week 4

## Risks acknowledged

  - SOP v3 prompts have NOT been LLM-tested before this commit. Dry run 
    on 5 papers will reveal whether Qwen2.5-14B actually follows the rules.
  - cleaning_v2 boundary detection relies on Docling identifying 
    section_header correctly. Some PDFs (especially older / OCR-only) 
    may have ALL_CAPS headers Docling labels as "text" not "section_header".
    Fallback: if no body_start match, start_idx = 0 (no harm).
  - 14B model may produce inconsistent rule-following on long documents. 
    If rules are violated > 10% of the time, need to consider:
      a) shorter prompts (move rules to system message)
      b) chain-of-thought variant (let model "check" before output)
      c) post-processing scrubber (regex remove **bold**, page refs)
